# Run Behaviour Specification

## Purpose

`Run` submits one generated batch (`seedBase`) to Slurm using metadata stored in SQLite.

## Inputs

- `seedBase` selected from generated batches.
- Generated run metadata from `generated_runs`.
- Slurm/TOPAS settings from configuration:
  - `Tsebt.Topas.Executable`
  - `Tsebt.Slurm.Partition`
  - `Tsebt.Slurm.CpusPerTask`

## Paths

Run folder:

```text
{AppRoot}/{Paths.Runs}/{seedBase}
```

Generated by Run:

```text
{AppRoot}/{Paths.Runs}/{seedBase}/run_manifest.tsv
{AppRoot}/{Paths.Runs}/{seedBase}/run_batch.slurm
```

## Manifest format

Header:

```text
task_id\tnode_name\trun_id\tinput_file_path\tlog_file_path
```

Each row contains:

- task id (1-based)
- node name (resolved from configured nodes by node digit)
- run id
- input file path
- log file path (`runs/{seedBase}/{runId}.log`)

## Preflight checks

Preflight verifies:

- generated runs exist
- batch status is `Generated`
- no previous Slurm job id
- all input files exist
- run folder exists
- no output base collisions
- no output CSV collisions
- no output log collisions

If any check fails, submission is blocked.

## Script behavior

The Slurm script:

- defines `TOPAS` executable variable
- defines `MANIFEST` path variable
- resolves row by `SLURM_ARRAY_TASK_ID`
- reads `TASK_ID NODE_NAME RUN_ID INPUT_FILE LOG_FILE`
- runs:

```bash
srun --nodes=1 --ntasks=1 --nodelist="$NODE_NAME" "$TOPAS" "$INPUT_FILE" > "$LOG_FILE" 2>&1
```

## Submission flow

`submitRun` performs:

1. Load batch details.
2. Block already-submitted batches (`Submitted/Running/Completed/Failed` with job id).
3. Run preflight.
4. Build manifest and script text.
5. Write manifest and script files.
6. Execute `sbatch {scriptPath}` with `ProcessStartInfo`.
7. Parse Slurm job id from output (`Submitted batch job 12345`).
8. Update SQLite batch metadata (`run_status`, `slurm_job_id`, paths, timestamp).
9. Update generated runs status to `Submitted`.

If `sbatch` fails, files remain for debugging and batch status is not marked `Submitted`.

## UI behavior notes

- Server stores full absolute paths.
- Run wizard UI displays AppRoot-relative paths for readability where possible.
